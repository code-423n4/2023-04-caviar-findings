## Gas optimization
### [G] Move calculations that can be performed outside the loop to the outside. 
Performing the same calculation repeatedly will increase gas usage.

### [G] Adding the payable keyword to the constructor
If you make the constructor payable, the initial check of msg.value == 0 is removed and you save 13 opcodes at deployment time without any security risk.

### [G] Functions that are expected to revert when called by normal users can be marked as payable
So e.g., when using a function modifier such as onlyOwner, the function would revert when a normal user tries to call it. By marking the function as payable, it reduces the cost of gas for the granted caller, since the compiler does not check to see if a payment has been made.
